# 06. Handoff: optional booker group + store booker_name from JWT

> **Status: 🔲 Not started** (as of 2026-07-08)

## Goal

Two related changes to bookings:

1. **Allow booking without a scout group.** Some users have no `/j26-scoutid-sync/groups/<id>` entry in their JWT. Today `booking.create` returns **403 "No scout group found in token"** for them (`server/src/server/web/booking.gleam:65-71`). Instead, a booking must be allowed with no group.
2. **Store `booker_name`.** Persist the booker's name from the JWT (`User.name`, which always exists) as a new column, distinct from the user-entered `responsible_name`.

To support (1), make `booker_group_id` and `booker_group_name` **nullable** in the DB and everywhere they flow.

### Semantics (decided)

- `booker_name` = `user.name` from the verified JWT. **NOT NULL** (name always present). It is the logged-in booker; `responsible_name` remains the separately-entered "responsible adult" from the booking form. Keep both.
- `booker_name` is set **on create only** — it records who made the booking and is not part of `BookingInput` (the update form). Do **not** add it to `update_booking`.
- `booker_group_id` / `booker_group_name` are `Option`: `Some` when the token has a group (name via existing `web.group_id_to_name`, a placeholder — see the `TODO(group-name)` in `web.gleam`), `None` otherwise.

## Steps

Work in a worktree if you like the established flow; otherwise a branch off `main`. `.env.sh` at repo root has `DATABASE_URL` (postgres on :5433, docker `j26booking-db` is up).

1. **Migration.** Scaffold with `cd server && gleam run -m cigogne new --name booking_optional_group_and_booker_name` (never hand-create the file), then fill:
   ```sql
   --- migration:up
   ALTER TABLE booking ALTER COLUMN booker_group_id DROP NOT NULL;
   ALTER TABLE booking ALTER COLUMN booker_group_name DROP NOT NULL;
   ALTER TABLE booking ADD COLUMN booker_name TEXT NOT NULL DEFAULT '';
   ALTER TABLE booking ALTER COLUMN booker_name DROP DEFAULT;
   --- migration:down
   ALTER TABLE booking DROP COLUMN booker_name;
   ALTER TABLE booking ALTER COLUMN booker_group_name SET NOT NULL;
   ALTER TABLE booking ALTER COLUMN booker_group_id SET NOT NULL;
   --- migration:end
   ```
   (The `DEFAULT ''` then drop is so the NOT NULL add succeeds if any rows exist; seeded rows get a real value in step 6.) Apply: `gleam run -m cigogne up`.

## Gotchas

- **Squirrel maps NULL → `Option(T)` automatically** from the schema; you don't annotate it — but the DB column must actually be nullable before regenerating, so do the migration (step 1) before `gleam run -m squirrel` (step 2).
- The shared `booking_decoder` has a hand-written UUID-validation failure branch with a `dummy` Booking — update that constructor too or it won't compile.
- Don't add `booker_name` to `BookingInput` or `update_booking`'s SET clause — it's derived from the JWT at create time, not user-editable.
- Follow the project skills: `squirrel-conventions` (SQL/codegen), `gleam-conventions` (Option vs Result — here Option is correct because null group is valid *data*, not a failure). Run `gleam format` before committing; conventional commit scope `db`/`api`.
- Existing auth tests (`server/test/server/web_test.gleam`) don't touch bookings and should stay green.
